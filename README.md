ПРИЛОЖЕНИЕ №2: КАРТА ПРОИСШЕСТВИЙ
========================================================================
========================================================================
1. НАЗНАЧЕНИЕ ПРИЛОЖЕНИЯ
------------------------------------------------------------------------
Карта происшествий — это приложение для отметки дорожных ситуаций на карте. Пользователь
может выбрать тип происшествия, нажать на нужное место на карте и добавить метку. Такое
приложение имитирует простой сервис для фиксации ДТП, ремонтов дорог, пробок и опасных
участков.

2. ОСНОВНЫЕ ВОЗМОЖНОСТИ
------------------------------------------------------------------------
• отображение карты Google Maps;
• поиск места на карте;
• получение текущего местоположения пользователя;
• выбор типа происшествия;
• добавление метки нажатием на карту;
• просмотр списка происшествий в отдельном окне;
• переход к выбранной метке из списка;
• удаление происшествий и маркеров;
• сохранение данных между запусками приложения.

3. ИНТЕРФЕЙС
------------------------------------------------------------------------
Интерфейс выполнен в тёмных цветах. На главном экране большую часть занимает карта.
Сверху расположена панель управления: поле поиска, кнопка «Найти», список выбора типа
происшествия и кнопка открытия списка.

Такой вариант удобен, потому что карта остаётся основным элементом приложения.
Пользователь сразу видит местность и может быстро поставить метку в нужной точке.

4. ДОБАВЛЕНИЕ ПРОИСШЕСТВИЯ
------------------------------------------------------------------------
Перед добавлением пользователь выбирает тип происшествия: ДТП, ремонт дороги, пробка,
перекрытие или опасный участок. Затем он нажимает пальцем на нужное место на карте.
После этого приложение создаёт объект происшествия, сохраняет его и рисует маркер.

Для разных типов используются разные цвета маркеров. Это помогает визуально отличать ДТП
от ремонта или пробки.

5. СПИСОК ПРОИСШЕСТВИЙ
------------------------------------------------------------------------
Список открывается в отдельном полноэкранном окне без создания новой Activity. Вверху
находится заголовок и кнопка «Назад». Ниже отображаются карточки происшествий. В каждой
карточке есть тип, координаты, кнопка «Открыть» и кнопка «Удалить».

Если нажать «Открыть», окно закрывается, а карта перемещается к выбранной метке. Если
нажать «Удалить», происшествие удаляется из списка, с карты и из сохранённых данных.

6. РАБОТА С ГЕОЛОКАЦИЕЙ И КАРТОЙ
------------------------------------------------------------------------
Приложение запрашивает разрешение на доступ к геолокации. Если пользователь разрешает
доступ, на карте включается кнопка текущего местоположения. Для поиска места
используется Geocoder: пользователь вводит название, приложение получает координаты и
перемещает камеру карты к найденной точке.

Для отображения карты используется Google Maps SDK. Метки создаются через MarkerOptions,
а дополнительная зона вокруг происшествия отображается с помощью CircleOptions.

7. ХРАНЕНИЕ ДАННЫХ
------------------------------------------------------------------------
Все происшествия сохраняются в SharedPreferences. Для каждой записи хранится уникальный
id, тип происшествия, широта и долгота. При повторном запуске приложения данные
загружаются и маркеры снова отображаются на карте.

8. ИТОГ
------------------------------------------------------------------------
Проект показывает работу с картами, геолокацией, поиском мест, маркерами и локальным
хранением данных. Главная идея приложения — быстро отмечать дорожные происшествия и
управлять ими через карту и список.
